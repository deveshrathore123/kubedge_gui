Languages for Creating a Static Website

A static website is built using the following technologies:

  • HTML (HyperText Markup Language)
  • CSS (Cascading Style Sheets)
  • JavaScript (optional, for interactivity)

Languages for Creating a Dynamic Website

A dynamic website involves both frontend and backend technologies. Here's a list of commonly used languages:

Frontend (Client-Side)

  • HTML
  • CSS
  • JavaScript
  • TypeScript (superset of JavaScript)
  • React.js, Angular, Vue.js, Svelte (Frontend frameworks, optional)

Backend (Server-Side)

  • PHP
  • Python (with frameworks like Django, Flask)
  • Node.js (with frameworks like Express)
  • Ruby (with Ruby on Rails)
  • Java (with frameworks like Spring)
  • C# (with ASP.NET)
  • Go
  • Rust (with Rocket, Actix)
  • Perl
  • Elixir (with Phoenix)
  • Scala (with Play)

Additional Technologies

  • GraphQL (alternative to REST APIs)
  • WebSocket (for real-time communication)

Databases

Choose the type of database for your website:

Relational Databases:

  • MySQL
  • PostgreSQL
  • SQLite
  • MariaDB
  • Oracle Database
  • Microsoft SQL Server
  • IBM Db2

Non-Relational (NoSQL) Databases:

  • MongoDB (Document-based)
  • Cassandra (Wide-column store)
  • CouchDB (Document-based)
  • Redis (Key-value store)
  • Elasticsearch (Document store, search engine)
  • Neo4j (Graph-based)
  • DynamoDB (Key-value store by AWS)
  • RavenDB (Document-based)
  • Couchbase (Document-based, key-value)
  • Amazon Aurora (Hybrid: relational and NoSQL)
